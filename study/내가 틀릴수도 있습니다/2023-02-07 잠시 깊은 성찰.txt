2023-02-07 잠시 깊은 성찰

"완벽한 이론 학습만을 추구하는 오만에 빠진건 아닐까?"
"기초 공사를 핑계로 단순 암기만을 추구하고 있는 것은 아닌가?"
"스스로의 능력치를 억제하는 오만에 빠지고 있지는 않은가?"
"배워도 모르는게 반이다. 계속 앞으로 나아가면서 부족한걸 채우자."

"과거의 부족함을 채우려고 하지 말고, 현재 나아가는데 있어 부족함을 채우는데 최선을 다하자."


HTML+CSS+JS 영역에서 이미 나는 분명히 웹 페이지를 만들고자 하면 만들어 낼 수 있다.
- API를 활용해 볼 수도 있고, MDN 레퍼런스를 활용할 수도 있다.
- 컴포넌트를 조합하듯 필요한 기능을 적재 적소에 배치할 수 있다.
- 빠르게 만들어보자. 하고 집중하면, 분명히 만들어 낼 수 있다.
- 무엇보다, 부족하지만 이미 WEB FE 개발자로서의 경험치는 있다.
- 무엇보다 W3 계속 확인해야된다. CSS3 은 지속적으로 레퍼런스가 업데이트가 된다.
- 모던 브라우저도 계속 바뀌어나가고 기술은 계속 가속화된다.
- 하드코딩에 미쳐서 모든 코드를 외우고 타자 연습마냥 뽑아내려고만 하는 오만에 빠져있는것은 아닌가?
=> 레퍼런스 총동원 해서 페이지 하나 빌드 하자 그냥.



프로그래밍 언어 차원에서, 나는 주력언어를 정하지 않고 수박겉핥기처럼 핥았다.
- 주력언어를 Java 로 정했고, 나아가고 있다.
- 자료형, 형변환, 변수, 함수, 메소드, 외부 라이브러리 활용에 대해 이미 알고있다.
- 방법론은 이미 알고있고, 적용하고자 하면 충분히 적용할 수 있다.
- 필요한 레퍼런스 자료가 어디에 있는지 알고 있다.
- 적합한 오픈 소스를 탐색해서 적용하고자 하면 적용할 수 있다.
- 암기하고자 하는 자료형 별 바이트 길이, 부동소수점 이론에 대해 너무 강박을 느끼는건 아닐까?
부호부-지수부-가수부 로 이어지는 내용을 이미 알고있다면, 더 나아가며 알아보면 되는 것은 아닐까?
- "네이티브 앱" "웹 앱" "하이브리드 앱" 의 차이와 어떤 언어를 적용하면 되는지 이미 알고 있다.
- 특정 언어 까는 글을 보기 전에, 내가 그 언어를 제대로 써본 적은 있나?
=> 계획 한 것을 구현 해 나가면서, 응용 해 나가면서 붙여야 되지 않을까?



서버사이드 개발 차원에서는, 배워야 할 것이 수도 없이 많다.
- 활용가능한 업체가 많았고, 내가 배우지 않은게 크다.
=> Java 하고 있으니까 Spring 으로 나아 갈 예정.



DB 차원에서, DB를 유지하고 관리하는 법을 경험하지 않는건 아닐까?
- 무엇보다 DB에 대한 이론 학습이 부족한 것은 사실이다.
- 쿼리, 스키마에 대해서 완벽하게 설명할 수 없다고 느끼는 것 또한 사실이다.
	DB에 적합한 사용자를 생성하고, 적당한 권한을 부여할 수 있다.
	DB를 오픈하면 무엇이 스키마이고, 무엇이 쿼리인지 안다.
	적합한 값을 수정하고, 관리를 시도 해 볼 수도 있다.
	스키마는 테이블이 되고, 쿼리는 한 행이 된다. 라고 말할 수는 있다.
- SELECT 문, FIND 문 과 같은 SQL 문법에 대해서 이론으로 암기하려 할 뿐, 계속 경험하고 있지 않는건 아닐까?
=> 적합한 DB를 결정하고, 지속적으로 유지하는 DB를 구축하는 방법에 대해서 고민하고 있지 않는건 아닐까?



OS 차원에서, 이미 알고 있는 내용이 많은데 필요할 때 바로바로 적용하는 능력이 없는게 아닐까?
- CRUD 에 대한 개념을 알고있다.
- rwx 에 대한 개념을 알고있고, chown 을 통해 적합한 퍼미션을 부여하고, 재설정 시킬 수 있다.
- 사용자 권한, 사용자 그룹에 대해서 알고 있고, 권한을 부여할 수 있다.
- 전문가라고 말하기는 부족하지만, CLI 환경을 기본적으로 헤쳐나갈 수 있는 명령어는 알고 있다.
- 관리자 권한을 사용하는 방법도, sudo 를 적당히 활용하는 방법도 알고 있다.
- File Descriptor 도, 스케줄링도 내가 암기하고 이해하려고만 하지 꾸준히 활용하지 않아서는 아닐까?
- 최소한 스케줄링에 대해서는 펜으로 그려봐야 했던건 아닐까?
- 방화벽 규칙 설정 할 줄 아는데, 필요할 때 고민 안하고 안 해 버릇 하는건 아닐까?
=> 내가 필요한 상황일 때, 내가 알고 있는 지식을 총동원해서 매일매일 적용하는 것이 최고의 학습 아닐까?
=> 나아가면서 디버깅이 필요해질 때, 페이징을 한 번 씩 생각해 보는 것도 좋지 않을까?



자료구조와 알고리즘 차원에서, 내가 알고리즘을 제대로 학습했다고 말할 수 있을까?
- 트리를 그리는 것도, 노드에 관한 것도, 간선에 관한 것도 알고 있다.
- 서브트리가 생성되는 기준에 대해서도 알고있다. 부모노드와의 연결을 끊었을때, 자식노드의 수 만큼 서브트리가 생성된다는 것 또한 알고있다.
- BFS 탐색을 하고, DFS 탐색을 하고, 전위탐색, 중위탐색, 후위탐색을 하는 것을 알고 있다.
- 문자열 탐색 시 KMP 알고리즘을 활용해 볼 수 있다는 것 또한 알고있다.
- 솔직히 지금 내가 이를 설명할 수 있는 것은, 지난 2주간 복습하고 노트하는 것을 경험해서 라고 생각한다.
- 솔직하게 내가 단 한 번이라도 자료구조와 알고리즘을 프로그래밍 언어로서 제대로 구현해 본 적이 있을까?
- 대학시절을 포함해서 직접 진지하게 생각해보거나, 도식화를 해보거나, 프로그래밍 언어로서 표현해본 노력이 있었을까?
=> 단순 암기가 아니라 문서를 보고, 노티 하고, 도식화 해보고, 언어로 구현해보면 되지 않을까?



네트워크 차원에서, 내가 네트워크 이론을 제대로 학습했다고 말할 수 있을까?
통신의 역사를 암기 할 뿐, 통신의 본질인 "주고 받는다" 를 스스로가 구현하지 않고 있는 것은 아닐까?
- OSI 7-Layer 가 응용 계층, 세션 계층, ... 통신계층, 물리계층 이라고 말하기 이전에,
내가 단순 암기하려고 OSI-7Layer 삼각형만 들여다 보지는 않았나?
- 최소한 내가 OSI 7-Layer에 대해서 직접 펜을 잡고 그려 본 적은 있었나?
- 응용 계층에서, 세션계층에서, 통신계층에서, 물리 계층에서 어떠한 일이 일어나는지 이해하고, 직접 적용해보다 보면 닿는게 아닐까?
- 네트워크 용어에 대해서 내가 단어를 암기하기 바빴지 제대로 이해하려고 해본 적은 있나?
- 진심으로 송신을 하고 수신을 받는다 이외에 제대로 이해하고 있는 내용이 있나?
- 5G가 NR 이라는 것, 802.11 규약에 관한 것 등등. 단어만 암기하려다 기억이 안난나고 화내다 포기한 것은 아닌가?
- 프로토콜은 통신을 하기 위한 약속이라는 것 이외에 더 이해를 시도해보려고 한 적은 있나? TCP/IP, UDP 등에 대해 제대로 이해하려고 해본 적은 있나?
- L3, L7 라우터, 스위치에 대해서 암기하려고 했지, 제대로 이해해보려 도식화 해본 적 있었나?
- WAN, LAN 에 대해 알고있고, 라우터의 역할, 스위치의 역할, 브릿지의 역할, 허브의 역할에 대해 알고있다면 최소한 연결해서 구상해봐야 되는 것 아닌가?
- 페이로드 밖에 안떠오른다 라고 하기 이전에, 직접 계속 이해하려고 시도해봐야 되는 것은 아닐까?
- 진심으로, RJ-45 이론으로 외우고 있는 것 보다 직접 까보니까 지금 당장 가위로도 딸 수 있는 것 처럼.
	WAN 을 도식화 해보고, LAN 을 도식화 해보고, 구상하고 적용하면서 나아가면 될 것 같지 않나?
- telnet 이 떠오를 때, telnet 을 켜보고 직접 연결을 시도 해 본 적은 없지 않나? 그냥 해보면 익혀질 것 같지 않나?
=> 시간 정해서 WAN 을 직접 그려보며 천천히 이해하면서 학습하는게 최고의 통신 학습 아닐까?
=> 원격 접속도 직접 접속을 해보고 핑도 날려보고 나아가서 클라이언트 쪽 소켓도 짜봐야 늘지 않을까?



아무리 적어봐도 더 넓은 세상을 이미 알고, 날 준비가 되었는데 날개를 접고있는 아기새 같아.
실패를 무서워하지 않는 척 다른 길로 새고 있는 어린 아이 같아.
책 도돌이표 암기 돌리지 말고, => 에 적어놓은 것들 다 직접 해보자.

영어랑 일본어가 됐다 안됐다 하는 것도.
연속수학이 안풀려진다고 느끼는 것도.

다시 복습해서 덧붙여지는 것도 사실이지만, 내가 완벽주의에 빠져서 응용하지 않으려는게 반 이상 아닐까?
틀리면 뭐 어때. 하고 나아가고, 매 순간 생기는 의문과 해봐야겠다 싶은 것을 실제로 행하는 것이 성장일텐데.
과거의 부족함을 채우려는게 아니라, 현재 성장하는데 있어서의 부족함을 채우는 것이 성장하는 마인드일텐데.
앞으로 나아갈 줄 알고, 방향을 알면서 겁먹어서 스스로 얼어있는게 아닐까?

어느 순간 부터 "거대한 장벽을 깨고 나아가는 것" 을 즐기지 않게 된 것이 아닐까?
진짜로 미적분 안 풀린다고 느낄 때 마다 "덧셈 뺄셈 부터 다시해야지" 하고
영어 Grammer 안된다고 느낄 때 마다 Voca 암기하고 있고
기초공사를 한다는 핑계로 계속 스스로의 자존감을 낮추고 있는 것은 아닐까?

가진 능력치를 총 동원하고, 최선을 다해서 미적분을 이해하려고 하고, 문제를 해결하려고 하면 과연 못 풀까?
영문 일간지를 최선을 다해서 읽어보려고 노력하면 과연 읽고 이해하는 것을 못 할까?
진심으로 매 순간이 최선이었고, 집중이었을까? 최선을 다해보긴 했을까?
장벽에 부딪힐 때 마다 기초가 부족하다며 스스로를 상처주는게 아닐까? 그게 남 탓으로 번지기도 하는게 아닐까?

그게 너무 오랜 시간동안 지속되어서 습관이 되어버린게 아닐까?
