- 프로그램의 어원, 자동화에 관해서
	프로그램의 어원: 연주회에서 음악을 연주할 시간들을 미리 정해놓는데,
		정해진 시간에 맞게 순서대로 연주하는 것을 "프로그램" 이라고 부르기로 함.
	이후로 "프로그램" 은 "정해진 시간 순서에 맞춰 순차적으로 연관된 일들이 진행되는 것" 이 됨.

	자동화: 사용자가 목적에 따라 순서대로 정한 필요한 일들을 컴퓨터가 시간 순서에 맞춰 순서대로 처리하는 것.
		- "내가 하고자 하는 목표" 를 정하고,
		- 목표에 필요한 일을 순서대로 정해서
		- 컴퓨터가 처리하게 만드는 것.
		이를 "자동화" 라고 함.

- Java 를 통한 IOT 개발 학습
	- IOT 기기를 작동시킨다고 가정한 오픈 소스 삽입.
	- 기존 프로젝트에 오픈 소스를 드래그 앤 드롭 -> Copy 시킴.
	- import 명령어를 활용한 패키지 및 객체 활용
		- import 명령어 사용하여 패키지 하위의 객체 불러오기 진행.
			import org.opentutorials.iot.elevator;
			// *iot 까지가 패키지 명, elevator 는 해당 패키지에 있는 객체 명.
		- import 명령어가 없을 경우 패키지명부터 객체까지 그대로 입력하면 사용 가능.
		- import 명령어로 객체를 불러왔을 경우 객체이름 만으로도 사용 가능.
	- new 키워드를 통한 객체 인스턴스 생성 및 인자 전달
		Elevator myElevator = new Elevator("JAVA APT 507");
	- 이후 중복되는 인자를 문자열 변수로 변경함.
		- 문자열 연산도 추가로 활용.
			Lighting myLighting = new Ligiting(id + " / Floor Light")

** 실제 아두이노 등 IOT를 작동시키는 오픈소스 패키지라면, 불러와서 활용했을 때 진짜 엄청나겠구나. 생각 함.

- Java 모듈 기술자 선행 학습
	- Java 9 이후 부터는 awt 등 외부 객체 사용 시 오류 발생함.
		즉, 기본 객체 사용 시에도 모듈 기술자에 정의 해줘야 함
	- 프로젝트 루트 경로에 module-info.java 파일을 생성하고, 모듈 정보를 기술하게 됨.
		module 모듈명 {
			종속 정보 
			외부 참조 등을 기술하게 됨.
		}
	- requires 키워드로 Java 기본 객체 종속됨을 기술할 수 있음.
	- 기본 모듈에는 java.base 모듈과 java.se 모듈이 있음.
		- java.base 모듈은 Java 11, java.se 모듈은 기본 java 모듈임.
		- java.base 모듈 하위에는 java.math, java.util 등이 포함됨.
		- java.se 모듈 하위에는 java.display 객체가 포함됨.
		- java.display 객체 하위로 java.awt 객체가 포함 됨.
	- 즉 java.se 모듈 -> java.display -> java.awt 순으로 상속이 이루어짐.
- awt 객체 사용 시 오류가 발생해서, 최상위 모듈인 java.se 모듈을 종속한다고 기술함.
